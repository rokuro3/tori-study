## 回答サマリー

### アプリケーション構成

| サービス | 役割 | ローカルでの扱い |
|----------|------|------------------|
| **Vercel** | フロントエンドホスティング | `npm run dev` で代替 |
| **Supabase** | 認証・DB・ストレージ | 本番と同じものを使用 |

### ローカル環境でのテスト手順

```bash
# 1. 依存パッケージのインストール
cd /root/toriStudy/app
npm install

# 2. 開発サーバー起動
npm run dev

# 3. ブラウザでアクセス
# http://localhost:3000
```

### 本番環境との差異

| 項目 | ローカル | 本番 |
|------|----------|------|
| URL | http://localhost:3000 | https://tori-study.vercel.app |
| HTTPS | なし | あり |
| ビルドモード | 開発（ホットリロード） | 本番（最適化済み） |
| Supabase | **共有** | **共有** |

### 重要な注意点

⚠️ **Supabaseは本番環境と共有**

ローカルでも本番のSupabaseプロジェクトに接続しているため：
- ユーザー登録データは本番にも反映
- スコアデータも本番DBに保存
- テストと本番データが混在する可能性

**解決策**: 開発用に別のSupabaseプロジェクトを作成することも可能

## 補足情報

### Supabaseを分離したい場合

開発環境と本番環境でSupabaseを分けるには：

1. 新規Supabaseプロジェクトを作成（例: `tori-study-dev`）
2. 同じSQLでテーブルを作成
3. `.env.local` に開発用の接続情報を設定
4. `.env.production` または Vercel環境変数に本番用を設定

これにより、開発中のテストデータが本番に影響しなくなります。
